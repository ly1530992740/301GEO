# GEO 后续优化 TODO

本文记录本轮 Topify 对标改造中，第一版已用规则、AI 估算或数据结构先支撑，但后续需要继续完善的事项。

## 1. 手动监控增强

当前状态：

1. 报告已经保存 Prompt 组和 Prompt 级结果。
2. `integrated_data.json` 已包含 `prompt_runs`、`geo_visibility_summary`、`brand_visibility_metrics`。
3. 历史报告重新打开后可以看到本次快照指标。

待完善：

1. 在历史报告列表增加“用同一组 Prompt 重新监控”按钮。
2. 复用历史 `question_discovery.questions`，不重新生成 Prompt。
3. 保存同品牌、同 Prompt 组的多次监控快照。
4. 增加趋势图：GEO 可见度总分、平均推荐位置、情绪分、Prompt 成功率。
5. 后续再考虑自动定时监控、失败重试和通知。

## 2. 五平台声量改真实数据

当前状态：

1. 百度、搜狗、360、抖音、小红书声量仍由 Qwen、豆包、元宝、DeepSeek 估算。
2. 报告中已标注“AI 估算，不等同于真实平台接口统计”。
3. 估算数据带 `confidence`、`notes`、`source_urls`。

待完善：

1. 接入真实百度搜索结果数量。
2. 接入搜狗、360 的真实搜索结果数量。
3. 调研抖音、小红书是否有可用官方接口或第三方数据源。
4. 将 `metric_type=multi_ai_estimated_count` 替换为真实来源，例如 `baidu_api_count`、`xiaohongshu_api_count`。
5. 报告中区分“真实接口统计”和“AI 估算补充”。

## 3. 情绪分从规则升级为模型评分

当前状态：

1. 情绪分基于 AI 推荐理由和回答摘要进行规则评分。
2. 默认中性 50 分，正面词加分，风险词扣分。
3. 这是“AI 内容 + 规则评分”的第一版，不是独立模型深度判断。

待完善：

1. 增加 Qwen 或 DeepSeek 的结构化情绪评分。
2. 让模型输出正面依据、负面依据、风险提示和可信度。
3. 引入行业词典，例如医美、法律、保健品等高风险行业需要更严格的负面词。
4. 对“没出现的品牌”不要显示默认情绪分，应显示“未进入 AI 推荐，无法判断”。

## 4. Prompt 生成质量提升

当前状态：

1. 默认调用 Qwen 生成 5 个真实用户问题。
2. Qwen 失败时使用规则兜底。
3. 本地服务会带地区，招商/B2B/C端会生成不同问法。

待完善：

1. Prompt 生成需要做去偏：不要因为客户品牌画像出现而把客户品牌相关词写进问题。
2. 增加 Prompt 分类：消费推荐、价格比较、资质/安全、口碑评价、服务流程。
3. 增加 Prompt 权重：越主流的问题权重越高。
4. 历史监控时固定 Prompt，避免每次问题变化导致趋势不可比。

## 5. 引用来源和文章抓取

当前状态：

1. AI 返回的 `citation_urls` 和搜索来源会被保存。
2. 程序会抓取部分文章正文做内容定位分析。
3. 抓取失败会记录错误，但不等于该品牌没有内容。

待完善：

1. 增加浏览器渲染抓取，处理部分 JS 页面。
2. 增加重试、超时和站点黑名单。
3. 对抓取失败的文章，使用标题、摘要、域名做轻量分析。
4. 引用来源按域名分类：官网、媒体、论坛、百科、社交平台、内容平台。
5. 增加“机会来源”：竞品被引用但客户未覆盖的来源。
